# 產品需求文件 (Product Requirements Document - PRD)

## 專案名稱：任務管理系統 (Task Management System)

---

## 1. 專案概述

### 背景與動機
在日常工作與生活中，人們常因事情繁雜而遺忘重要任務，或是難以掌握目前的工作進度。為了解決個人時間管理與任務追蹤的需求，我們規劃打造一個輕量化、直覺且高效的「任務管理系統」。本系統旨在提供一個乾淨明瞭的介面，協助使用者隨手記錄待辦事項、即時掌握任務狀態，進而提升日常工作效率與專注度。

### 目標用戶
- 需要管理日常待辦事項的個人用戶（如學生、上班族、自由職業者）。
- 追求簡潔介面、不希望被複雜專案管理工具（如 Jira、Trello）干擾的輕量使用者。

### 核心價值主張
- **極簡體驗**：無需學習成本，開啟網頁即可快速新增、管理任務。
- **狀態清晰**：一目了然的完成狀態，以及靈活的篩選機制，幫助使用者專注於當前最重要的事。
- **輕量高效**：系統加載快速，操作反應即時，不拖泥帶水。

## 2. 功能需求

以下為系統之主要功能需求與對應的使用者故事：

### 主要功能清單與使用者故事

1. **新增任務**
   - **使用者故事**：作為一名**個人用戶**，我希望能夠**在介面上輸入任務名稱並快速新增任務**，以便**隨手記錄突然想到的待辦事項**。
   - **詳細描述**：輸入框應有 placeholder 提示，並支援按下 Enter 鍵或點擊「新增」按鈕進行提交。

2. **標記任務完成**
   - **使用者故事**：作為一名**個人用戶**，我希望能夠**一鍵勾選或標記某項任務為「已完成」**，以便**知道哪些工作已經處理完畢，並獲得成就感**。
   - **詳細描述**：任務旁設有核取方塊 (Checkbox) 或狀態切換按鈕，標記完成後，任務文字應有刪除線或其他明顯的視覺區隔，且可隨時取消標記（回復為未完成）。

3. **刪除任務**
   - **使用者故事**：作為一名**個人用戶**，我希望能夠**刪除不再需要或輸入錯誤的任務**，以便**保持任務清單的乾淨與整潔**。
   - **詳細描述**：每條任務旁應有「刪除」按鈕（垃圾桶圖示或文字），點擊後即時從清單中移除該任務。

4. **顯示所有任務清單**
   - **使用者故事**：作為一名**個人用戶**，我希望能夠**在一個頁面上看到我所有的待辦任務**，以便**通盤掌握當前的工作量與項目**。
   - **詳細描述**：主畫面以列表形式呈現所有任務，並依據建立時間或完成狀態進行基本排序。

5. **依完成狀態篩選**
   - **使用者故事**：作為一名**個人用戶**，我希望能夠**依據「全部」、「未完成」或「已完成」狀態篩選任務**，以便**快速聚焦在仍需處理的工作，或回顧已達成的成果**。
   - **詳細描述**：介面上提供切換標籤 (Tabs) 或按鈕，讓使用者快速切換三種檢視模式。

## 3. 非功能需求

### 技術限制
- **後端框架**：使用 Python Flask 框架。
- **前端模板**：使用 Jinja2 模板引擎進行 HTML 動態渲染。
- **樣式設計**：使用原生 CSS (Vanilla CSS) 進行視覺設計與排版，確保極佳的客製化與美觀度，避免使用過於繁重的框架。
- **資料儲存**：使用輕量級 SQLite 資料庫，儲存任務的相關資料。

### 效能與安全考量
- **頁面載入速度**：單頁應用程式般的順暢體驗，首頁載入時間應在 1 秒以內。
- **資料驗證**：後端需對輸入的任務名稱進行長度與空白防錯驗證（防止空任務或過長任務導致排版損壞）。
- **SQL 注入防護**：使用 SQLAlchemy ORM 或參數化查詢，防止 SQL Injection 攻擊。
- **RWD 響應式網頁設計**：介面需完美適應手機、平板與桌上型電腦，確保在行動裝置上也能順暢操作。

## 4. MVP (最小可行性產品) 範圍

為了快速驗證產品並提供核心價值，功能優先級規劃如下：

### Must Have (必須有 — MVP 核心)
- 新增任務功能。
- 顯示所有任務清單。
- 標記任務完成/未完成（狀態變更）。
- 刪除任務功能。
- 基礎的 SQLite 資料庫儲存，確保重新整理網頁後資料不會消失。

### Should Have (應該有)
- 依完成狀態篩選任務（全部、未完成、已完成）。
- 任務名稱的輸入防錯驗證（如：不可為空、字數上限）。
- 手機板與桌機板的響應式佈局 (RWD)。

### Nice to Have (可以有 — 未來擴充)
- 任務的優先等級設定（高、中、低）與標記顏色。
- 任務的截止日期 (Due Date) 與過期提醒。
- 分類標籤 (Tags / Categories) 功能。
- 使用者註冊與登入系統（多用戶隔離）。
